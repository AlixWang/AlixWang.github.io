###第一部分 基本概念

######1.1 严格模式

ECMAScript 5引入了严格模式（strict mode）的概念。严格模式是为Javascript定义了一种不同的解析和执行模型。在严格模式下，ECMAScript 3 中的一些不确定的行为将得到处理。对某些不安全的操作也会抛出错误。

启用方法

"use strict"

在函数内部的上方包含着条编译指示，也可以制定函数在严格模式下执行。

function dosomthing(){
"use strict"
//函数体    
}

在严格模式下，JavaScript的执行结果会有很大的不同，支持严格模式的浏览器包括IE10+、FF4+、SF5.1+、Opera12+、Chrome。

######1.2变量

ECMAScript的变量是松散类型的，可以用来保存任何类型的数据.
①定义变量时使用Var关键字定义的是具有特定作用域的变量，在函数内定义的变量则其作用域就是函数的作用域；当省略Var关键字时，不论其位于何位置，其作用域都是全局的。所以在定义变量时为了便于管理和产生意外的错误建议使用Var关键字。

notes 在严格模式下如果给未声明变量赋值会抛出ReferenceError：** is not defined*.错误

②JavaScript中变量没有块级作用域

for(var i=0; i<10 ; i++){
    alert(i)
}
alert(i);//仍然可以弹出i的值

notes 使用ECMAScript6 中的let关键字可以模拟块级作用域

for(let i=0; i<10; i++){
    alert(i);
}
alert(i);//i未定义

#####1.3数据类型

ECMAScript中有5种简单数据类型（也称为基本数据类型）Undefined、Null、Number、String和一种复杂数据类型Object。

① Typeof操作符

② Undefined类型

已经声明但为赋值的变量为Undefined
将Undefined赋值给变量也为Undefined

③Null类型

Null类型表示的是一个空的对象指针

④Boolean 类型

BooLen 类型是ECMAScript 中使用得最多的一种类型，该类型只有两种值true和false。

notes：Boolean类型的字面值true和false是区分大小写。即True和False都不是Boolean的值只是标识符。

将其他类型值转换为Boolean值的方法，调用Boolean()函数。

var message = "hellow";
var messageBoolean  = Boolean(message);    //true

在实际应用时基本都是自动转换的，例如在if语句中判断时。

⑤Number类型

ECMAscript中的Number类型使用IEEE754格式来表示，整数和浮点数值（浮点数值在某些语言中也被称为双精度数值）

十进制 intNum = 55 //整数
八进制 必须以0开头，然后是八进制数字序列（007）

var octalNum1 = 070        //八进制的56
var octalNum2 = 079        //无效的八进制---解析为79
var octalNum3 = 08        //无效的八进制---解析为8

十六进制 开头以0x表示

var hexNum = 0xA     //十六进制的10

notes：在进行算数运算时所有以八进制和十六进制表示的数值都将被转换成十进制数值。

var octalNum = 073;    //59
var hexNum = 0xA;    //10
console.log(octalNum + hexNum);    //69

浮点值

所谓的浮点值，就是数值中必须包含一个小数点，并且小数点后面至少有以为数字。虽然小数点后面可以没有数字，但不推荐这种写法。

var floatNum1 = 1.1;
var floatNum2 = 0.1;
var floatNum3 = .1;    //有效但是不推荐

由于保存浮点数对内存开销比保存整数大得多所以ECMAScript会对可以转换为整数的浮点数转换为整数。

var floatNum1 = 1.;    //小数点后面没有数字———>解析为1
var floatNum2 =10.0;    //整数—————>解析为10

对于极大或极小的数字采用科学计数法。

var floatNum = 3.125e7         //31250000

关于浮点数值不能进行精确计算，因为ECMAScript进行浮点计算时需要先将其转换为二进制表示，然后再进行计算，所以会产生误差（在操作符节进行细说）

数值范围

Number.MIN_VALUE        //可存储的最小值
Number.MAX_VALUE        //可存储的最大值
/*
    不同的浏览器可能有不同的最大最小值但根据我的测试Nodejs的V8引擎、FF、IE都是一样的
*/

当超过最大或最小范围时就会返回Infinity值，所以ECMAScript有函数siFinate()来检测到底是否是有效值。

NAN(Not a Number)

NAN本身有两个非同寻常的特点。首先，任何设计NAN的操作都会返回NaN。这个特性在多步计算中可能导致问题。其次，NaN与任何值都不相等。所以ECMAScript定义了isNaN()函数。

alert( NaN == NaN )        // false

alert( isNaN( NaN ) )    // true
alert( isNaN( 10 ) )    // false
alert( isNaN( "10" ) )    // false
alert( isNaN( "blue" ) )    // true
alert( isNaN( true ) )    // false

isNaN() 也可以用来检测对象，基于对象调用jsNaN()时，会首先调用对象的valueOf()方法，然后确定该方法返回值是否可以转换为数值。如果不能，则基于这个返回值在调用toString()方法，再测试返回值。

数值转换

有三个函数可以把非数值转换为数值：Number()、parseInt()、parseFloat()。

Number()的转换规则

  • .如果是Boolean值，则true和false将分别转换为0和1；
  • .如果数字则是本身；
  • .如果null值则是0；
  • .如果是undefined返回NaN；
  • .如果是字符串则遵循
    1 .只包含数字则直接返回对应的数值，去除字符串中的前导零；
    2 .包含有效的浮点值，则转换为对应的浮点值去除前导零；
    3 .包含有效的十六进制值（0x123），则转换为对应的十进制值；
    4 .如果字符串为空，则转换为0；
    5 .如果字符串中包含除上述格式之外的字符，则将其转换为NaN。

parseInt()的转换规则

  • .忽略前导0和前导空格；
  • .字符串为空返回NaN；
  • .如果开头字符不是数字则返回NaN；
  • .浮点值会解析为整数值；

  • .可以解析数字格式（八进制，十进制，十六进制）；

    var num1 = parseInt( “1234blue” ) //1234
    var num2 = parseInt( “ “ ) //NaN
    var num3 = parseInt( “0xA” ) //16
    var num4 = parseInt( 22.5 ) //22
    var num5 = parseInt( “070” ) //56
    var num6 = parseInt( “70” ) //70
    var num7 = parseInt( “blue1234” ) //NaN
    parseint()最好指定后面的基数这样就会避免错误的发生

paseFloat()的转换规则

  • 字符串中的第一个小数点有效
  • 只解析10进制值
